class

bitmap% : class?

  superclass: object%

bitmap%对象是基于像素的图像，可以是单色、彩色，也可以是带alpha通道的彩色。另请参见make-bitmap、make-platform-bitmap、make-screen-bitmap (来自racket/gui/base)、make-bitmap in canvas% (来自racket/gui/base)和可移植性和位图变体。

位图有一个备份比例（backing scale），即当位图用作绘图目标或当位图绘制到另一个上下文中时，与位图的绘图单位相对应的像素数。例如，在Mac OS上，当主监视器处于“视网膜（Retina）”模式时，make-screen-bitmap将返回一个支持比例为2.0的位图。在Windows和Unix上，屏幕位图的备份比例对应于系统范围的文本比例（请参见(part ("(lib scribblings/gui/gui.scrbl)" "显示分辨率"))）。单色位图的支持比例始终为1.0。

位图可以通过file/convertible协议转换为'png-bytes。

make-bitmap、make-monochrome-bitmap和read-bitmap函数创建bitmap%实例，但它们也比直接使用带bitmap%的make-object更受欢迎，因为函数的重载程度较低，默认情况下启用alpha通道。另见可移植性和位图变体。

提供width和height时：创建一个新位图。如果monochrome?为真，位图是单色的；如果monochrome?为#f和alpha?为真，则位图具有alpha通道；否则，位图是没有alpha通道的彩色。backing-scale参数设置位图的备份比例（backing scale），如果monochrome为真，则必须为1.0。位图的初始内容为“空（empty）”：全部为白色，如果位图具有Alpha通道，则为零Alpha。

如果提供了in：从文件格式创建位图，其中kind指定格式。有关详细信息，请参见load-file。backing-scale参数设置位图的备份比例，以便位图的大小（由get-width和get-height报告）是位图大小的ceiling（从in除以backing-scale）；如果位图是单色或加载了一个遮罩，备份比例必须为1.0。

提供bits字节字符串时：从位值数组创建单色位图，其中每个bits中的字节指定8位，并添加填充位，以便每个位图行从字符边界开始。1位值表示黑色，0表示白色。如果width乘以height大于bits长度的8倍，an exn:fail:contract exception is raised。

Changed in version 1.1 of package draw-lib: 添加backing-scale可选参数。

在unscaled?为#f时，生成与get-argb-pixels in bitmap-dc%相同的结果，但位图不必选入DC（即使位图选入另一个DC，附加为按钮标签等，此方法也有效）。

如果位图的备份比例（backing scale）不是1.0，并且unscaled?为真，则结果与忽略备份比例的位图像素相对应。在这种情况下，x、y、width和height有效地以像素为单位，而不是以绘图为单位。

Changed in version 1.1 of package draw-lib: 添加#:unscaled?可选参数。

method

(send a-bitmap get-loaded-mask) → (or/c (is-a?/c bitmap%) #f)

返回与此位图一起存储的掩码位图。

当加载带有'gif/mask或'unknown/mask且文件包含透明“彩色”的GIF文件时，将生成一个遮罩位图来标识透明像素。遮罩位图是单色的，其中白色像素的加载位图是透明的，黑色像素是其他地方。

当一个PNG文件加载了'png/mask或'unknown/mask并且该文件包含一个mask或alpha通道时，会生成一个mask位图来标识该mask或alpha通道。如果文件包含只有极值的mask或alpha通道，则mask位图为单色，否则为灰度（表示反转的alpha通道）。

当用'xpm/mask或'unknown/mask加载XPM文件时，将生成一个遮罩位图，以指示设置了哪些像素。

当使用'unknown/alpha和类似模式加载位图时，透明度信息由阿尔法通道表示，而不是由遮罩位图表示。

与alpha通道不同，绘制例程不会自动使用遮罩位图。可以显式提取和提供遮罩位图作为遮罩（例如，作为draw-bitmap的第六个参数）。当将位图保存为'png时，如果遮罩的尺寸与save-file相同，则保存文件将使用遮罩位图。当位图是控件标签时，也会自动使用遮罩位图。

(send a-bitmap get-width) → exact-positive-integer?

获取位图的宽度（以绘图单位表示）（与备份比例的像素相同）为1.0。

method

(send a-bitmap load-file in    
  [kind    
  bg-color    
  complain-on-failure?]) → boolean?
  in : (or/c path-string? input-port?)
  kind : 
(or/c 'unknown 'unknown/mask 'unknown/alpha
      'gif 'gif/mask 'gif/alpha
      'jpeg 'jpeg/alpha
      'png 'png/mask 'png/alpha
      'xbm 'xbm/alpha 'xpm 'xpm/alpha
      'bmp 'bmp/alpha)
 = 'unknown
  bg-color : (or/c (is-a?/c color%) #f) = #f
  complain-on-failure? : any/c = #f

从in读取的文件格式加载位图，除非位图是由make-platform-bitmap、make-screen-bitmap或make-bitmap in canvas%（在这种情况下，an exn:fail:contract exception is raised）产生。如果位图由bitmap-dc%对象或控件使用，则不会加载图像数据。位图会更改其大小和深度以匹配加载的图像。如果在读取文件格式时遇到错误，则只有如果complain-on-failure?为真才会引发异常（不是默认值）。

kind参数指定文件的格式：

  • 'unknown ——检查文件以确定其格式；创建没有alpha通道的单色或彩色位图；

  • 'unknown/mask ——类似于'unknown，但请参见get-loaded-mask；

  • 'unknown/alpha ——类似于'unknown，但如果位图是彩色的，则它具有alpha通道，并且图像文件中的透明度记录在alpha通道中；

  • 'gif ——加载一个GIF位图文件，创建一个彩色位图；

  • 'gif/mask ——类似'gif，但请参见get-loaded-mask；

  • 'gif/alpha ——类似'gif，但带有alpha通道；

  • 'jpeg ——加载JPEG位图文件，创建彩色位图；

  • 'jpeg/alpha ——类似于'jpeg，但带有alpha通道

  • 'png ——加载PNG位图文件，创建彩色或单色位图；

  • 'png/mask ——类似于'png，但请参见get-loaded-mask；

  • 'png/alpha ——类似于'png，但始终使用彩色和alpha通道；

  • 'xbm ——加载X位图（XBM）文件；创建单色位图；

  • 'xbm/alpha ——类似于'xbm，但使用alpha通道创建彩色位图；

  • 'xpm ——加载XPM位图文件，创建彩色位图；

  • 'xpm/alpha ——类似于'xpm，但带有alpha通道；

  • 'bmp ——加载Windows位图（BMP）文件，创建彩色位图；

  • 'bmp/alpha ——类似于'bmp，但带有alpha通道

XBM图像总是作为单色位图加载。没有遮罩或阿尔法通道的1位灰度PNG也加载为单色位图。任何其他格式的图像总是作为彩色位图加载。

对于PNG和BMP加载，如果bg-color不是#f，则在加载图像时将其与文件的alpha通道或遮罩（如果有的话）组合；在这种情况下，不会生成单独的遮罩位图，并且alpha通道填充位图，即使为格式指定了'unknown/mask，也不会指定'png/mask。如果将格式指定为'unknown或'png，并且未指定bg-color，则将参考PNG文件以获取用于加载的背景色，如果文件中未指示背景色，则使用白色。

在所有PNG加载模式中，当文件提供gamma值时应用gamma校正，否则不应用gamma校正。当前显示的gamma因子由SCREEN_GAMMA环境变量（如果定义了）确定。如果preference和environment变量都未定义，则使用平台特定的默认值。

创建位图后，仅当位图的备份比例为1.0时，才能使用load-file。

(send a-bitmap ok?) → boolean?

method

(send a-bitmap save-file name    
  kind    
  [quality    
  #:unscaled? unscaled?]) → boolean?
  name : (or/c path-string? output-port?)
  kind : (or/c 'png 'jpeg 'xbm 'xpm 'bmp)
  quality : (integer-in 0 100) = 75
  unscaled? : any/c = #f

将位图写入命名文件或输出流。

kind参数确定了创建的文件类型，其中一个：

  • 'png ——保存PNG文件；

  • 'jpeg ——保存JPEG文件；

  • 'xbm ——保存X位图（XBM）文件；

  • 'xpm ——保存XPM位图文件；

  • 'bmp ——保存Windows位图文件。

quality参数仅用于保存为'jpeg，在这种情况下，它指定图像精度（高质量与bitmap%对象的内容更精确地匹配）和大小（低质量更小）之间的权衡。

当保存为'png时，如果get-loaded-mask返回的位图与此位图的大小相同，则灰度版本将作为alpha通道包含在PNG文件中。

保存为'png的单色位图（不带遮罩位图）将生成一个1位灰度PNG文件（当使用load-file读取时，该文件将创建一个单色bitmap%对象）。

如果位图的备份比例不是1.0，那么在保存之前，它会有效地转换为每个绘图单位的单个像素，除非unscaled?是真的。

Changed in version 1.1 of package draw-lib: 添加#:unscaled?可选参数。
