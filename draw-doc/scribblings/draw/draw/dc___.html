interface

dc<%> : interface?

一个dc<%>对象是用于绘制图形和文本的绘图上下文。它以通用的方式表示输出设备；例如，一个画布有一个绘图上下文，一个打印机都也一样。

method

(send a-dc cache-font-metrics-key) → exact-integer?

返回一个整数，如果不是0，则对应于一个特定类型的设备和比例因子，以便文本范围信息（来自 get-text-extent、 get-char-height等）相同。该键在所有dc<%>实例中都有效，即使在不同的类中也是如此。

一个0结果表示a-dc的当前配置不属于公共类别，因此没有可用于缓存文本扩展数据块信息的密钥。

method

(send a-dc clear) → void?

清除绘图区域（使用由get-background确定的当前背景色填充）。也可以参见《 erase》。

method

(send a-dc copy x y width height x2 y2) → void?

  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
  x2 : real?
  y2 : real?

将由绘图上下文的x、y、 width和height定义的矩形区域复制到由 x2和y2指定的位置处的同一图形上下文。源区域和目标区域可以重叠。

Changed in version 1.12 of package draw-lib: 允许源和目标重叠。

method

(send a-dc draw-arc x    
  y    
  width    
  height    
  start-radians    
  end-radians) → void?
  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
  start-radians : real?
  end-radians : real?

绘制一个逆时针的圆弧，椭圆的一部分，该椭圆内接在由 x （左（left））、y （顶（top））、width和height指定的矩形中。圆弧从start-radians 指定的角度开始（0是3点钟，半圆周率是12点钟），然后逆时针继续到end-radians。如果 start-radians和end-radians相同，则绘制一个完整的椭圆。

当前画笔用于圆弧。如果当前画笔不是透明的，则用于填充由圆弧加上延伸到内接椭圆中心的线（未绘制）所限定的楔体。如果笔和画笔都是不透明的，则在用笔绘制圆弧之前，楔块中会用画笔填充。

楔形和弧形相交，因此它们之间没有空间，但是楔形和弧形之间的精确重叠是平台和尺寸特定的。通常，画笔和笔绘制的区域重叠。在非平滑或对齐模式下，通过缩小边界椭圆的宽度和高度来调整轮廓路径，缩放后，用一个绘图单位除以对齐比例。

method

(send a-dc draw-bitmap source    
  dest-x    
  dest-y    
  [style    
  color    
  mask]) → boolean?
  source : (is-a?/c bitmap%)
  dest-x : real?
  dest-y : real?
  style : (or/c 'solid 'opaque 'xor) = 'solid
  color : (is-a?/c color%)
   = (send the-color-database find-color "black")
  mask : (or/c (is-a?/c bitmap%) #f) = #f

显示source位图。dest-x和dest-y参数采用DC坐标。

对于颜色位图，将忽略绘图样式和颜色参数。对于单色位图，draw-bitmap使用样式和颜色参数的方式与画笔使用其样式和颜色设置绘制单色点画的方式相同（有关详细信息，请参见brush%。

如果提供了一个mask位图，则其宽度和高度必须与source及其ok?必须返回true，否则an exn:fail:contract exception is raised。source位图和mask位图可以是同一个对象，但如果绘图上下文是一个bitmap-dc%对象，则两个位图必须与目标位图不同，否则an exn:fail:contract exception is raised。

mask对绘图的影响取决于mask位图的类型：

  • 如果mask位图是单色的，则仅当遮罩位图包含黑色像素（与style无关，后者控制如何处理单色source的白色像素）时，才会在目标dc<%>中绘制。

  • 如果mask位图是带alpha通道的颜色，则其alpha通道将用作绘制source的遮罩，并且其颜色通道将被忽略。

  • 如果mask位图是没有alpha通道的颜色，则对给定像素的颜色分量进行平均，以获得该像素的逆alpha值。特别是，如果mask位图为灰度，则每个遮罩像素的黑度控制绘制像素的不透明度（即，遮罩充当一个倒置的alpha通道）。

DC的当前画笔、当前笔和当前文本对位图的绘制方式没有影响，但如果DC具有比例，则位图将被缩放，并且DC的alpha设置确定绘制像素的不透明度（结合source的alpha通道、任何给定的mask以及当source为单色时color的alpha分量）。

对于post-script-dc%和 pdf-dc%输出，来自source中的alpha通道、来自mask或来自color的不透明度四舍五入为完全透明或不透明。

如果位图绘制成功，结果是#t，否则是#f（可能是因为位图的ok?方法返回#f）。

也可参见draw-bitmap-section。

method

(send a-dc draw-ellipse x y width height) → void?

  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))

绘制包含在具有给定左上角和大小的矩形中的椭圆。当前笔用于轮廓，当前画笔用于填充形状。如果笔和画笔都是不透明的，则在用笔绘制轮廓之前，椭圆将用画笔填充。

画笔填充和笔轮廓相吻合，这样它们之间就没有空间了，但是填充和轮廓之间的精确重叠是平台和尺寸特定的。因此，画笔和笔绘制的区域可能部分重叠。在非平滑或对齐模式下，轮廓的路径在缩放后通过一个绘图单位除以对齐比例来调整椭圆的宽度和高度。

method

(send a-dc draw-line x1 y1 x2 y2) → void?

  x1 : real?
  y1 : real?
  x2 : real?
  y2 : real?

从一点到另一点画一条线。当前笔用于绘制线条。

在非平滑模式下，点对应于像素，线条覆盖起点和终点。对于一个宽度大于1的笔，画的线是以起点和终点为中心的。

另请参见set-smoothing了解有关'aligned平滑模式的信息。

(send a-dc draw-lines points

使用点的列表points绘制线，向每个点添加xoffset和yoffset。一个配对被视为一个点，其中配对的car是x值，cdr是y值。当前笔用于绘制线条。

method

(send a-dc draw-path path    
  [xoffset    
  yoffset    
  fill-style]) → void?
  path : (is-a?/c dc-path%)
  xoffset : real? = 0
  yoffset : real? = 0
  fill-style : (or/c 'odd-even 'winding) = 'odd-even

绘制给定 dc-path%对象的子路径，将xoffset和yoffset添加到每个点。（有关路径和子路径的一般信息，请参见dc-path%）当前笔用于将路径绘制为直线，当前画笔用于填充路径边界区域。

如果笔和画笔都是不透明的，则在使用笔绘制轮廓之前，将使用画笔填充路径。填充和轮廓相交，因此它们之间没有空间，但是填充和轮廓之间的精确重叠是平台和尺寸特定的。因此，画笔和笔绘制的区域可能重叠。通常情况下，笔在路径上居中，以非平滑模式向左和向下取整。

fill-style参数指定填充规则：'odd-even或'winding。在'odd-even模式下，如果一个点被奇数个子路径循环包围，则该点被视为封闭在路径中。在'winding模式下，如果一个点被多余或少于顺时针子路径循环包围，而不是逆时针子路径循环包围，则认为该点被包围在路径内。

另请参见set-smoothing了解有关'aligned平滑模式的信息。

method

(send a-dc draw-polygon points    
  [xoffset    
  yoffset    
  fill-style]) → void?
  points : 
(or/c (listof (is-a?/c point%))
      (listof (cons/c real? real?)))
  xoffset : real? = 0
  yoffset : real? = 0
  fill-style : (or/c 'odd-even 'winding) = 'odd-even

使用点列表points绘制一个填充多边形，向每个点添加xoffset和yoffset。一个配对被视为car是x值，而cdr是y值。多边形自动闭合，因此第一个点和最后一个点可以不同。当前笔用于绘制轮廓，当前画笔用于填充形状。

如果钢笔和画笔都是不透明的，则在用钢笔绘制轮廓之前，多边形将用画笔填充。填充和轮廓相交，因此它们之间没有空间，但是填充和轮廓之间的精确重叠是平台和形状特定的。因此，画笔和笔绘制的区域可能重叠。通常情况下，笔在多边形线上居中，以非平滑模式向左和向下取整。

fill-style参数指定填充规则：'odd-even或'winding。在'odd-even模式下，如果一个点被奇数圈包围，则该点被视为封闭在多边形内。在'winding模式下，如果一个点被多于或少于逆时针圈的顺时针圈包围，则认为该点被包围在多边形内。

另请参见 set-smoothing了解有关'aligned平滑模式的信息。

method

(send a-dc draw-rectangle x y width height) → void?

  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))

绘制具有给定左上角和大小的矩形。当前笔用于轮廓，当前画笔用于填充形状。如果笔和画笔都是不透明的，则在用笔绘制轮廓之前，矩形中会填充画笔。

在非平滑或对齐模式下，当笔的大小为0或1时，填充会精确地重叠整个轮廓。通常，在非平滑或对齐模式下，通过缩小矩形的宽度和高度来调整轮廓路径，缩放后，通过对齐比例分割一个绘图单位。

另请参见set-smoothing了解有关'aligned平滑模式的信息。

method

(send a-dc draw-rounded-rectangle x    
  y    
  width    
  height    
  [radius]) → void?
  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
  radius : real? = -0.25

绘制具有给定左上角和给定大小的矩形。角是使用给定半径的四分之一圆。当前笔用于轮廓，当前画笔用于填充形状。如果笔和画笔都是不透明的，则矩形将被填充。在用钢笔画轮廓之前用刷子。

如果radius为正数，则该值将用作圆角的半径。如果radius为负数，则绝对值将用作矩形最小尺寸的 比例。

如果radius小于-0.5或大于width或height的一半，an exn:fail:contract exception is raised。

画笔填充和笔轮廓相吻合，这样它们之间就没有空间了，但是填充和轮廓之间的精确重叠是平台和尺寸特定的。因此，画笔和笔绘制的区域可能部分重叠。在非平滑或对齐模式下，轮廓的路径在缩放后通过一个绘图单位除以对齐比例来调整矩形的宽度和高度。

另请参见set-smoothing了解有关'aligned平滑模式的信息。

method

(send a-dc draw-text text    
  x    
  y    
  [combine?    
  offset    
  angle]) → void?
  text : string?
  x : real?
  y : real?
  combine? : any/c = #f
  offset : exact-nonnegative-integer? = 0
  angle : real? = 0

使用当前文本字体、当前文本前景色和背景色在指定点绘制文本字符串。对于未汇总的文本，指定的点用作绘制字符的左上角起点（例如，如果绘制了“W”，则该点大致是“W”中左上角像素的位置）。旋转的文本围绕该点旋转。

text字符串从offset字符开始绘制，一直绘制到text或第一个空字符结束。

如果combine?是#t，那么text可以用相邻字符组合到连字符来测量，Unicode组合字符作为单个连字符，用紧排，用从右到左的字符呈现等。如果是combine?为#f，则结果与单独测量每个字符的结果相同，并且忽略Unicode控制字符。

字符串逆时针旋转angle弧度。如果angle不是零，则文本始终以透明模式绘制（请参见 set-text-mode）。

DC的当前画笔和当前笔设置对文本的绘制方式没有影响。

有关绘制文本的大小信息，请参见get-text-extent。

另请参见set-text-foreground、set-text-background和set-text-mode。

method

(send a-dc end-doc) → void?

结束文档，仅在绘制到打印机、PostScript、PDF或SVG设备时相关。

对于相关设备，如果在文档不是以start-doc启动时调用end-doc，当页面当前是由start-page启动而不是以start-page结束时调用，或者文档已经结束时调用end-page，则会引发异常。

method

(send a-dc end-page) → void?

仅当绘图到打印机、PostScript、PDF或SVG设备时，才结束一页。

对于相关设备，如果end-page在当前未由start-page启动页面时调用，则会引发异常。

method

(send a-dc erase) → void?

对于具有alpha通道的绘图上下文，erase将所有alphas设置为零。同样，对于透明画布，erase会擦除所有绘图，以允许背景窗口显示。对于没有alpha通道或透明度的其它绘图上下文，erase将绘图上下文填充为白色。

method

(send a-dc flush) → void?

为canvas<%>输出调用flush in canvas<%>方法，并且对其他类型的绘图上下文无效。

method

method

(send a-dc get-text-extent string 
  [font 
  combine? 
  offset]) 
 → 
(and/c real? (not/c negative?))
(and/c real? (not/c negative?))
(and/c real? (not/c negative?))
(and/c real? (not/c negative?))
  string : string?

结果是四个实数：

  • 文本的总宽度（取决于字体和文本）；

  • 字体的总高度（仅取决于字体）；

  • 字体基线到下伸器底部的距离（包括在高度中，仅取决于字体）；以及

  • 字体设计器添加到字体的额外垂直空间（包括在高度中，通常为零；仅取决于字体）。

返回的宽度和高度定义了一个矩形，它保证在绘制时包含文本字符串，但不一定紧密匹配。根据字体设计人员的想法和平台特定的字体缩放机制，绘制字符串左侧、右侧、顶部和底部的一些未定义的像素可能是“空白”。

如果combine?是#t，那么text可以用相邻的字符组合到连字符，用Unicode组合字符作为一个单个连字符，用紧排，用从右到左的字符顺序绘制，等等。如果是combine?为#f，则结果与单独绘制每个字符的结果相同，并且忽略Unicode控制字符。

与大多数方法不同，此方法可以在不安装位图的情况下为bitmap-dc%对象调用。

method

(send a-dc set-smoothing mode) → void?

  mode : (or/c 'unsmoothed 'smoothed 'aligned)

为绘图启用或禁用抗锯齿平滑。（文本平滑不受此方法的影响，而是通过font%对象进行控制。）

平滑模式可以是'unsmoothed、'smoothed或'aligned。'aligned和'smoothed都是启用抗锯齿的平滑模式，而 'unsmoothed和'aligned都会调整绘图坐标以匹配像素边界。对于大多数绘制到屏幕或位图的应用程序来说，'aligned是最佳选择。

从概念上讲，整数绘图坐标对应于像素之间的边界，基于笔的绘图在给定的直线或曲线上居中。因此，在'smoothed模式下，使用笔宽1从(0, 10)到(10, 10)绘制2像素宽的线条，不透明度为50%。

在'unsmoothed和'aligned模式中，图形坐标根据图形上下文的对齐比例（alignment scale）被截断。具体来说，当对齐比例为1.0时，图形坐标将被截断为整数坐标。更一般地说，绘图坐标移向零，这样结果乘以对齐比例就成了积分。对于线条绘制，坐标将根据笔宽和对齐比例进一步移动，其中移动对应于笔宽的一半（减少到一个值，使其乘以对齐比例乘以2产生一个整数）。此外，对于通过draw-rectangle、draw-ellipse、draw-rounded-rectangle和 draw-arc绘制的笔，给定的宽度和高度均除以对齐比例后减小1.0。
