class

post-script-dc% : class?

  superclass: object%

  extends: dc<%>

post-script-dc%对象是PostScript设备上下文，可以在任何平台上写入PostScript文件。另请参见ps-setup%和pdf-dc%。

Be sure to use the following methods to start/end drawing:

  • start-doc

  • start-page

  • end-page

  • end-doc

Attempts to use a drawing method outside of an active page raises an exception.

另参见printer-dc%。

constructor

(new post-script-dc% 
    [[interactive interactive] 
    [parent parent] 
    [use-paper-bbox use-paper-bbox] 
    [as-eps as-eps] 
    [width width] 
    [height height] 
    [output output]]) 
 → (is-a?/c post-script-dc%)
  interactive : any/c = #t
  parent : (or/c (is-a?/c frame%) (is-a?/c dialog%) #f) = #f
  use-paper-bbox : any/c = #f
  as-eps : any/c = #t
  width : (or/c (and/c real? (not/c negative?)) #f) = #f
  height : (or/c (and/c real? (not/c negative?)) #f) = #f
  output : (or/c path-string? output-port? #f) = #f

如果interactive为真，则会为用户提供一个用于设置打印参数的对话框（请参见get-ps-setup-from-user）；结果配置将作为当前配置安装。如果用户选择打印到一个文件（在Windows和Mac OS上是唯一可能的），则会给出另一个对话框来选择文件名。如果用户在其中一个对话框中点击Cancel，那么ok?返回#f。

如果parent不是#f，它将用作配置对话框的父级窗口。

如果interactive为#f，则使用current-ps-setup返回的设置。如果get-file方法返回#f并output为#f，则仍会向用户显示一个文件对话框，在这种情况下，用户可能会单击Cancel，这样ok?返回#f。

如果use-paper-bbox为#f，则输出的PostScript边界框由width和height确定（使用ceiling向上取整）。如果use-paper-bbox不是#f，则边界框由当前纸张大小确定（由current-ps-setup指定）。当width或height为#f时，则相应的尺寸由纸张大小决定，即使use-paper-bbox为#f。

如果 as-eps是#f，则生成的PostScript不包含封装的PostScript（EPS）头，而是包含通用的PostScript头。只有当as-eps为#f时，才使用current-ps-setup指定的边距和转换因子。如果as-eps为真，则生成的PostScript包含一个标题，该标题将其标识为EPS。

如果output不是#f，则文件模式输出写入output。如果output为#f，则通过current-ps-setup或提示用户输入路径名来确定目标。当output为端口时，数据由post-script-dc%实例创建的线程写入port；如果关闭写入线程的保管器，则调用end-doc以thread-resume和(current-thread)作为第二个参数恢复端口写入线程。

另请参见ps-setup%和current-ps-setup。创建对象时，特定post-script-dc%对象的设置将固定为当前配置中的值（当interactive为真时，用户交互式地调整了这些值之后）。
