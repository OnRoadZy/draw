class

gl-config% : class?

  superclass: object%

一个gl-config%对象封装配置信息，用于一个OpenGL绘图上下文。使用gl-config%对象作为canvas%的一个初始化参数，或者提供给make-gl-bitmap。

constructor

(new gl-config%) → (is-a?/c gl-config%)

创建一个GL配置，该配置指示旧的OpenGL、双缓冲、大小为1的深度缓冲、无模具缓冲、无累积缓冲、无多采样和无立体声。

method

(send a-gl-config get-accum-size) → (integer-in 0 256)

报告配置请求的累积缓冲区大小（对于红、绿、蓝和alpha中的每一个），其中零表示不请求累积缓冲区。

method

(send a-gl-config get-depth-size) → (integer-in 0 256)

报告配置请求的深度缓冲区大小，其中零表示没有请求深度缓冲区。

method

(send a-gl-config get-double-buffered) → boolean?

报告配置是否请求双缓冲。

method

(send a-gl-config get-hires-mode) → boolean?

确定是否使用hires（雇用）模式。在Mac OS上，hires模式意味着创建的OpenGL上下文将能够访问完整的视网膜分辨率，并且不会被绘图系统缩放。在其他平台上，hires模式不起作用。

Added in version 1.5 of package draw-lib.

method

(send a-gl-config get-legacy?) → boolean?

确定是使用传统的“兼容性”OpenGL还是“核心”OpenGL。Mac OS（10.7及以上版本）和Linux（如果图形驱动程序支持）。目前支持核心OpenGL配置文件。

Added in version 1.2 of package draw-lib.

method

(send a-gl-config get-multisample-size) → (integer-in 0 256)

报告配置请求的多采样大小，其中零表示不请求多采样。

method

(send a-gl-config get-share-context)

 → (or/c #f (is-a?/c gl-context<%>))

返回与新创建的OpenGL绘图上下文共享某些对象（纹理、显示列表等）的gl-context<%>对象，或者#f表示没有设置。

另请参见set-share-context。

method

(send a-gl-config get-stencil-size) → (integer-in 0 256)

报告配置请求的模具缓冲区大小，其中零表示没有请求模具缓冲区。

method

(send a-gl-config get-stereo) → boolean?

报告配置是否请求立体声。

method

(send a-gl-config get-sync-swap) → boolean?

报告配置是否请求与屏幕刷新进行缓冲区交换同步。

Added in version 1.10 of package draw-lib.

method

(send a-gl-config set-accum-size on?) → void?

  on? : (integer-in 0 256)

调整配置以请求每个通道（红、绿、蓝和alpha）的特定累积缓冲区大小，其中零表示不请求累积缓冲区。

method

(send a-gl-config set-depth-size on?) → void?

  on? : (integer-in 0 256)

调整配置以请求特定的深度缓冲区大小，其中零表示不请求深度缓冲区。

method

(send a-gl-config set-double-buffered on?) → void?

  on? : any/c

是否调整配置以请求双缓冲。

method

(send a-gl-config set-hires-mode hires-mode) → void?

  hires-mode : any/c

是否调整配置以请求租用模式；请参见get-hires-mode。

Added in version 1.5 of package draw-lib.

method

(send a-gl-config set-legacy? legacy?) → void?

  legacy? : any/c

是否调整配置以请求传统模式；请参见get-legacy?

Added in version 1.2 of package draw-lib.

method

(send a-gl-config set-multisample-size on?) → void?

  on? : (integer-in 0 256)

调整配置以请求特定的多采样大小，其中零表示不请求多采样。如果多采样上下文不可用，将忽略此请求。

method

(send a-gl-config set-share-context context) → void?

  context : (or/c #f (is-a?/c gl-context<%>))

确定与新创建的OpenGL绘图上下文共享某些对象（纹理、显示列表等）的gl-context<%>对象，其中#f表示不应发生共享。

当上下文B与上下文A共享对象时，它也与与A共享其他所有上下文共享对象，反之亦然。

如果OpenGL实现不支持共享，则在创建新的上下文时，将有效地忽略上下文。所有版本的Mac OS都应该支持共享。在Windows和Linux上，共享是由WGL_ARB_create_context和GLX_ARB_create_context扩展分别提供的（而OpenGL3.2需要两者）。

method

(send a-gl-config set-stencil-size on?) → void?

  on? : (integer-in 0 256)

调整配置以请求特定的模具缓冲区大小，其中零表示不请求模具缓冲区。

method

(send a-gl-config set-stereo on?) → void?

  on? : any/c

是否调整配置以请求立体声。

method

(send a-gl-config set-sync-swap on?) → void?

  on? : any/c

调整配置以请求是否与屏幕刷新进行缓冲区交换同步。

Added in version 1.10 of package draw-lib.
