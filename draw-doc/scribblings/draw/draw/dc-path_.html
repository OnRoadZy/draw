class

dc-path% : class?

  superclass: object%

路径是由曲线定义的一组图形。路径可以与dc<%>对象的draw-path方法一起使用，以将路径的曲线绘制为直线、填充由路径曲线界定的区域，或者两者都使用。路径还可以与region%对象的set-path方法一起使用，以生成由路径曲线限定的区域。

一个路径包含零个或多个闭合子路径（closed sub-paths），可能还有一个开放子路径。某些dc-path%方法扩展了开放子路径（open sub-path），某些dc-path%方法关闭打开的子路径，而某些dc-path%方法添加了关闭的子路径。这种绘制公式的方法继承自PostScript[Adobe99]。

当一条路径被画成一条线时，一条封闭的子路径被画成一个封闭的图形，类似于一个多边形。打开的子路径以不相交的起点和终点绘制，类似的线以draw-lines in dc<%>绘制。

当路径被填充或用作区域时，打开的子路径（如果有）将被视为已关闭。路径的内容可以通过'even-odd或'winding来确定，该规则是在路径被填充或用于生成区域时选择的。

路径没有连接到任何特定的dc<%>对象，因此设置dc<%>原点或比例不会影响路径操作。相反，dc<%>的原点和比例在绘制路径或用于设置区域时使用。

constructor

(new dc-path%) → (is-a?/c dc-path%)

创建不包含子路径（也不包含开放子路径（open sub-path））的新路径。

method

(send a-dc-path append path) → void?

  path : (is-a?/c dc-path%)

将path的子路径添加到a-dc-path。path的闭合子路径（Closed sub-paths）作为闭合子路径添加到a-dc-path。如果两个路径都有一个开放子路径，则此路径的子路径将由给定路径的开放子路径扩展，并将一条线从该路径的当前终结点添加到给定路径的起始点。如果只有一条路径具有开放子路径，则它将成为（或保持）此路径的开放子路径。

method

(send a-dc-path arc x    
  y    
  width    
  height    
  start-radians    
  end-radians    
  [counter-clockwise?]) → void?
  x : real?
  y : real?
  width : real?
  height : real?
  start-radians : real?
  end-radians : real?
  counter-clockwise? : any/c = #t

用与椭圆截面相对应的曲线扩展或启动路径的开放子路径。如果width和height非负，则椭圆是由左上角为(x, y)且尺寸以高度为宽度的矩形限定的；如果width为负，则矩形的右边缘为x，椭圆宽度为(abs width)，而负height同样使y是椭圆的下边缘并且高度是(abs width)。对负width和height的支持有助于避免最终目标中对齐绘图的舍入问题，因为arc将其输入减少到一系列曲线。相比之下，draw-arc in dc<%>可以自动校正舍入。因为绘图模式是立即知道的。椭圆部分以角start-radians（0是三点钟，半π是12点钟）开始，并继续到角end-radians；如果counter-clockwise?为真，则弧从start-radians到end-radians为逆时针方向，否则为顺时针方向。

如果路径没有开放子路径，则新的子路径将从弧的起点开始。否则，圆弧会延伸现有的子路径，并且现有路径与到圆弧起点的直线连接。

method

(send a-dc-path close) → void?

关闭路径的开放子路径。如果路径没有开放子路径，an exn:fail:contract exception is raised。

method

(send a-dc-path curve-to x1 y1 x2 y2 x3 y3) → void?

  x1 : real?
  y1 : real?
  x2 : real?
  y2 : real?
  x3 : real?
  y3 : real?

使用点(x1,y1)和(x2,y2)作为控制点，使用贝塞尔曲线将路径的开放子路径扩展到给定点(x3,y3)。如果路径没有开放子路径，an exn:fail:contract exception is raised。

method

(send a-dc-path ellipse x y width height) → void?

  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))

关闭开放子路径，如果有，并添加一个闭合子路径，该子路径表示一个椭圆，该椭圆由一个左上角为(x, y)且尺寸为width乘以height。（这种方便的方法是通过close和arc来实现的。）

method

(send a-dc-path get-bounding-box) → 
real? real? real? real?

返回一个包含路径点的矩形。返回值是矩形的左侧、顶部、宽度和高度。

对于路径中的曲线，边界框封闭了两个控制点以及起点和终点。因此，边界框并不总是紧密地绑定路径。

method

(send a-dc-path get-datum) → 
(listof (listof vector?))
(listof vector?)

返回路径的矢量列表表示形式。第一个结果是一个列表，其中包含每个闭合子路径的列表，第二个结果是开放子路径的列表。如果路径没有开放子路径，则第二个结果是空列表。

表示子路径的每个列表都以表示路径起点的两个数字的向量开始。后面的每个元素要么是两个数字的矢量，表示一条将前一个点连接到新点的线，要么是六个数字的矢量，表示一条将前一个点连接到新点的曲线；在后一种情况下，矢量中的第五个和六个数字表示曲线的终点，第一个和第二个数字表示曲线的第一个控制点，第三个和第四个数字表示曲线的第二个控制点。

Added in version 1.8 of package draw-lib.

method

(send a-dc-path line-to x y) → void?

  x : real?
  y : real?

用一条线将路径的开放子路径扩展到给定点。如果路径没有开放子路径，an exn:fail:contract exception is raised。

method

(send a-dc-path lines points    
  [xoffset    
  yoffset]) → void?
  points : 
(or/c (listof (is-a?/c point%))
      (listof (cons/c real? real?)))
  xoffset : real? = 0
  yoffset : real? = 0

用一系列线将路径的开放子路径扩展到给定点。一配对被视为一个点，其中配对的car是x值，cdr是y值。如果路径没有开放子路径，an exn:fail:contract exception is raised。（此方便方法是根据line-to实现的。）

method

(send a-dc-path move-to x y) → void?

  x : real?
  y : real?

关闭开放子路径（如果有）后，将使用给定的初始点启动新的开放子路径。

method

(send a-dc-path open?) → boolean?

如果路径有开放子路径，则返回#t，否则返回#f。

method

(send a-dc-path rectangle x y width height) → void?

  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))

关闭开放子路径（如果有），并添加一个表示左上角为(x,y)且尺寸为width乘以height的矩形的闭合路径。（此方便方法是根据close、move-to和line-to实现的。）

method

(send a-dc-path reset) → void?

删除路径的所有子路径。

method

(send a-dc-path reverse) → void?

反转所有子路径中所有点的顺序。如果路径具有开放子路径，则起始点将成为终结点，并且对开放子路径的扩展将在此新的终结点上构建。当确定'winding模式下路径的内容时，反转闭合子路径会影响它与其他子路径的组合方式。

method

(send a-dc-path rotate radians) → void?

  radians : real?

调整路径中的所有点（包括所有子路径），围绕(0,0)逆时针旋转radians弧度。此调用不会旋转将来添加到路径的内容。

method

(send a-dc-path rounded-rectangle x    
  y    
  width    
  height    
  [radius]) → void?
  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
  radius : real? = -0.25

关闭开放子路径（如果有），并添加一个闭合子路径，该子路径表示左上角为(x y)且尺寸按width乘以height的圆角矩形。（这种方便的方法是在close、move-to、arc和line-to中实现的。）

如果radius为正数，则该值将用作圆角的半径。如果radius为负，则使用绝对值作为矩形最小尺寸的比例。

如果radius小于-0.5或大于width或height的一半，an exn:fail:contract exception is raised。

method

(send a-dc-path scale x y) → void?

  x : real?
  y : real?

调整路径中的所有点（包括所有子路径），将每个X坐标乘以x，每个Y坐标乘以y。用负数缩放会翻转相应轴上的路径。将来添加到路径中的内容不会通过此调用进行缩放。

method

(send a-dc-path text-outline font    
  str    
  x    
  y    
  [combine?]) → void?
  font : (is-a?/c font%)
  str : string?
  x : real?
  y : real?
  combine? : any/c = #f

关闭开放子路径（如果有），并使用font将闭合子路径添加到轮廓线str。文本的左上角位于x和y处。combine?参数启用紧排和字符组合，如同在draw-text in dc<%>一样。

method

(send a-dc-path transform m) → void?

  m : (vector/c real? real? real? real? real? real?)

通过应用m表示的转换调整路径中的所有点（包括所有子路径）。

有关用向量m表示的矩阵的信息，请参见get-initial-matrix。

method

(send a-dc-path translate x y) → void?

  x : real?
  y : real?

调整路径中的所有点（包括所有子路径），然后向右移动x，向下移动y。此调用不会转换将来添加到路径的内容。
