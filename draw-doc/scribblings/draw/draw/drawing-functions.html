procedure

(get-face-list [kind 
  #:all-variants? all-variants?]) 
 → (listof string?)
  kind : (or/c 'mono 'all) = 'all
  all-variants? : any/c = #f

返回当前系统上可用的字体名称列表。如果 kind是'mono，那么列表中只包含已知与monospace字体对应的面。

如果all-variants?是#f（默认值），那么结果是在更标准的术语中是字体系列名称列表，它与样式和权重选项组合在一起以到达一个表面；如果all-variants?为真，则结果包括族中每个可用表面的字符串。

procedure

(make-bitmap width    
  height    
  [alpha?    
  #:backing-scale backing-scale]) → (is-a?/c bitmap%)
  width : exact-positive-integer?
  height : exact-positive-integer?
  alpha? : any/c = #t
  backing-scale : (>/c 0.0) = 1.0

返回(make-object bitmap% width height #f alpha? backing-scale)，但是这个过程是首选的，因为它默认alpha?以一种更有用的方式。

另请参见make-platform-bitmap和可移植性和位图变体。

Changed in version 1.1 of package draw-lib: 添加了#:backing-scale可选参数。

procedure

(make-brush [#:color color    
  #:style style    
  #:stipple stipple    
  #:gradient gradient    
  #:transformation transformation    
  #:immutable? immutable?]) → (is-a?/c brush%)
  color : (or/c string? (is-a?/c color%)) = (make-color 0 0 0)
  style : 
(or/c 'transparent 'solid 'opaque
      'xor 'hilite 'panel
      'bdiagonal-hatch 'crossdiag-hatch
      'fdiagonal-hatch 'cross-hatch
      'horizontal-hatch 'vertical-hatch)
 = 'solid
  stipple : (or/c #f (is-a?/c bitmap%)) = #f
  gradient : 
(or/c #f
     (is-a?/c linear-gradient%)
     (is-a?/c radial-gradient%))
 = #f
  transformation : 
(or/c #f (vector/c (vector/c real? real? real?
                             real? real? real?)
                    real? real? real? real? real?))
   = #f
  immutable? : any/c = #t

创建brush%实例。与使用使用brush%的make-object相比，此过程提供了几乎等效的接口，但它还支持创建不可变的画笔（并在默认情况下创建不可变的画笔）。

当stipple为#f时，gradient为#f，transformation为#f，immutable?为真，并且color是color%对象或the-color-database中的字符串，结果画笔是通过the-brush-list的find-or-create-brush创建的。

刷子转换与get-transformation in dc<%>具有相同的表示和含义。

procedure

(make-color red green blue [alpha]) → (is-a?/c color%)

  red : byte?
  green : byte?
  blue : byte?
  alpha : (real-in 0 1) = 1.0

创建color%实例。此过程提供了一个与使用使用color%的make-object几乎等效的接口，但它创建了一个不可变的color%对象。

procedure

(make-monochrome-bitmap width height [bits]) → (is-a?/c bitmap%)

  width : exact-positive-integer?
  height : exact-positive-integer?
  bits : (or/c bytes? #f) = #f

如果bits为#f，则返回(make-object bitmap% width height #t)，否则返回(make-object bitmap% bits width height)。此过程优先于在bitmap%上使用make-object，因为它的重载程度较低。

 → (is-a?/c bitmap%)

  height : exact-positive-integer?

Changed in version 1.1 of package draw-lib: 添加#:backing-scale可选参数。

procedure

(read-bitmap in    
  [kind    
  bg-color    
  complain-on-failure?    
  #:backing-scale backing-scale    
  #:try-@2x? try-@2x?]) → (is-a?/c bitmap%)
  in : (or path-string? input-port?)
  kind : 
(or/c 'unknown 'unknown/mask 'unknown/alpha
      'gif 'gif/mask 'gif/alpha
      'jpeg 'jpeg/alpha
      'png 'png/mask 'png/alpha
      'xbm 'xbm/alpha 'xpm 'xpm/alpha
      'bmp 'bmp/alpha)
   = 'unknown/alpha
  bg-color : (or/c (is-a?/c color%) #f) = #f
  complain-on-failure? : any/c = #t
  backing-scale : (>/c 0.0) = 1.0
  try-@2x? : any/c = #f

返回(make-object bitmap% in kind bg-color complain-on-failure? backing-scale)，但这一程序是首选的，因为它以一种更有用的方式默认kind和complain-on-failure?。

如果try-@2x?为真，in是一个路径，kind不是/mask符号之一，然后读取位图检查是否存在与in匹配的文件，但名称中添加了"@2x"（如果有，在文件后缀之前）。如果存在"@2x"路径，则使用它而不是in，并且backing-store乘以2。

Changed in version 1.1 of package draw-lib: 添加#:backing-scale 和#:try-@2x?可选参数。
