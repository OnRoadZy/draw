1 概述

racket/draw库提供基于PostScript绘图模型的绘图API。它支持线条绘制、形状填充、位图复制、alpha混合和仿射转换（即缩放、旋转和平移）。

请参阅(part ("(lib scribblings/guide/guide.scrbl)" "classes"))了解Racket中的类和接口的介绍。

使用racket/draw进行绘图需要一个绘图上下文（drawing context）（DC），它是dc<%>接口的一个实例。例如，post-script-dc%类实现一个dc<%>用于绘制PostScript文件，而bitmap-dc%用于绘制位图。当对GUI使用racket/gui库时，get-dc方法返回一个dc<%>实例，以便在canvas窗口中绘制。

请参阅(part ("(lib scribblings/gui/gui.scrbl)" "canvas-drawing"))了解在GUI窗口中绘图的介绍。

1.1 线条和简单形状

要绘制到位图中，首先使用make-bitmap创建位图，然后创建一个bitmap-dc%以绘制到新位图中：

(define target (make-bitmap 30 30)) ; 一个30x30位图
(define dc (new bitmap-dc% [bitmap target]))

然后，使用诸如在DC上draw-line之类的方法绘制到位图。例如，序列

(send dc draw-rectangle
      0 10   ; 左上角（0，10），左上角向下10像素
      30 10) ; 30像素高和10像素宽
(send dc draw-line
      0 0    ; 开始（0，0），在左上角
      30 30) ; 画到（30，30），右下角
(send dc draw-line
      0 30   ; 从左下角的（0，30）开始
      30 0)  ; 画到（30，0），右上角

在较小的矩形顶部绘制一个“X”到target中。如果用(send target save-file "box.png" 'png)将位图保存到文件中，"box.png"包含图像

采用PNG格式。

draw-line之类的线条绘制操作使用DC的当前笔（pen）绘制线条。笔具有颜色、线条宽度和样式，其中笔样式包括'solid、'long-dash和'transparent。如draw-rectangle这样的封闭形状操作同时使用当前笔和DC的当前画笔（brush）。画笔具有颜色和样式，其中画笔样式包括'solid、'cross-hatch和'transparent。

在DrRacket中，您可以使用(require racket/gui)和(make-object image-snip% target)在DrRacket交互窗口中查看位图，而不是将target保存到从文件查看图像的文件中。

例如，在绘图操作之前，将画笔和笔设置为在带有蓝色细边框的绿色矩形上绘制厚的红色“X”：

(send dc set-brush "green" 'solid)
(send dc set-pen "blue" 1 'solid)
(send dc draw-rectangle 0 10 30 10)
(send dc set-pen "red" 3 'solid)
(send dc draw-line 0 0 30 30)
(send dc draw-line 0 30 30 0)

要绘制没有轮廓的填充形状，请将笔设置为'transparent模式（具有任何颜色和线条宽度）。例如，

(send dc set-pen "white" 1 'transparent)
(send dc set-brush "black" 'solid)
(send dc draw-ellipse 5 5 20 20)

默认情况下，位图bitmap-dc%绘制实体像素而不平滑形状的边界。要启用平滑，请将平滑模式设置为'smoothed或'aligned：

(send dc set-smoothing 'aligned)
(send dc set-brush "black" 'solid)
(send dc draw-ellipse 4 4 22 22) ; 稍微大一点

'aligned和'smoothed之间的差异与位图中像素的相对粗粒度有关。从概念上讲，绘图坐标对应于像素之间的线条，笔在线条上居中。在'smoothed模式下，在线条上绘制会使笔在线条的任一侧以一半的强度绘制，这会为1像素的黑色笔生成以下结果：

1.2 笔、画笔和颜色对象

DC接受pen%和brush%对象的set-pen和set-brush方法，将笔和画笔设置组合在一起。

(require racket/math)
 
(define no-pen (new pen% [style 'transparent]))
(define no-brush (new brush% [style 'transparent]))
(define blue-brush (new brush% [color "blue"]))
(define yellow-brush (new brush% [color "yellow"]))
(define red-pen (new pen% [color "red"] [width 2]))
 
(define (draw-face dc)
  (send dc set-smoothing 'aligned)
 
  (send dc set-pen no-pen)
  (send dc set-brush blue-brush)
  (send dc draw-ellipse 25 25 100 100)
 
  (send dc set-brush yellow-brush)
  (send dc draw-rectangle 50 50 10 10)
  (send dc draw-rectangle 90 50 10 10)
 
  (send dc set-brush no-brush)
  (send dc set-pen red-pen)
  (send dc draw-arc 37 37 75 75 (* 5/4 pi) (* 7/4 pi)))
 
(define target (make-bitmap 150 150))
(define dc (new bitmap-dc% [bitmap target]))
 
(draw-face dc)

get-pen和get-brush方法返回DC当前的笔和画笔，以便在临时更改它们进行绘图后恢复它们。

除了分组设置之外，pen%或brush%对象还包括其他设置，这些设置不能直接使用set-pen或set-brush。例如，笔或画笔可以有一个点画（stipple），点画是在绘制时使用的位图，而不是纯色。例如，如果"water.png"有图像

然后可以用read-bitmap加载并安装为blue-brush的点画：

(send blue-brush set-stipple (read-bitmap "water.png"))
(send dc erase)
(draw-face dc)

沿着类似的路线，color%对象允许您通过其红色、绿色和蓝色组件而不是内置的颜色名称指定颜色。由于color%初始化被重载，请使用make-object而不是new来实例化color%或使用make-color函数：

(define red-pen
  (new pen% [color (make-color 200 100 150)] [width 2]))
(send dc erase)
(draw-face dc)

1.3 变换

提供给图纸推荐的任何坐标或长度都由DC当前变换矩阵变换。变换矩阵可以缩放图像、在偏移位置绘制图像或旋转所有图形。可以直接设置转换，也可以使用scale、translate或rotate等方法进一步转换当前转换：

(send dc erase)
(send dc scale 0.5 0.5)
(draw-face dc)
(send dc rotate (/ pi 2))
(send dc translate 0 150)
(draw-face dc)
(send dc translate 0 -150)
(send dc rotate (/ pi 2))
(send dc translate 150 150)
(draw-face dc)
(send dc translate -150 -150)
(send dc rotate (/ pi 2))
(send dc translate 150 0)
(draw-face dc)

使用get-transformation方法获取DC的当前转换，并使用 set-transformation恢复保存的转换（或任何仿射转换）。

1.4 绘制路径

draw-line和draw-rectangle等绘图功能实际上是更一般的绘制路径操作的便利功能。draw-path操作采用一个路径（path），该路径描述要用笔绘制的一组线段和曲线，如果是闭合的一组线和曲线，则使用当前画笔填充。

dc-path%的实例包含一个路径。从概念上讲，路径有一个当前笔位置，可以通过诸如move-to、line-to和curve-to等方法来操作。move-to方法启动子路径，并对其进行line-to和curve-to延伸。close方法将笔从当前位置直线移动到起始位置，完成子路径，形成一个可以用画笔填充的闭合路径。一个dc-path%对象可以有多个封闭的子路径和一个最终的开放路径，其中开放路径只能用笔绘制。

例如，

(define zee (new dc-path%))
(send zee move-to 0 0)
(send zee line-to 30 0)
(send zee line-to 0 30)
(send zee line-to 30 30)

创建开放路径。用宽度为5的黑色笔和透明画笔绘制此路径将生成

1.5 文本

使用draw-text方法绘制文本，该方法需要一个字符串以便绘制及一个位置提供绘制文本的左上角：

(define text-target (make-bitmap 100 30))
(define dc (new bitmap-dc% [bitmap text-target]))
(send dc set-brush "white" 'transparent)
 
(send dc draw-rectangle 0 0 100 30)
(send dc draw-text "Hello, World!" 5 1)

用于绘制文本的字体由DC的当前字体决定。字体由font%对象描述，并用set-font安装。可使用set-text-foreground设置与笔或画笔分开的绘制文本的颜色。

(send dc erase)
(send dc set-font (make-font #:size 14 #:family 'roman
                             #:weight 'bold))
(send dc set-text-foreground "blue")
(send dc draw-rectangle 0 0 100 30)
(send dc draw-text "Hello, World!" 5 1)

要计算绘制文本将使用的大小，请使用get-text-extent，它返回四个值：总宽度、总高度、基线和总高度之间的差以及行中文本上方的额外空间（如果有）。例如，get-text-extent的结果可用于将文本定位到框的中心：

(send dc erase)
(send dc draw-rectangle 0 0 100 30)
(define-values (w h d a) (send dc get-text-extent "Hello, World!"))
(send dc draw-text "Hello, World!" (/ (- 100 w) 2) (/ (- 30 h) 2))

1.6 alpha通道和alpha混合

创建或erase位图时，内容为空。“没什么（Nothing）”和白（white）不一样，因为没有绘画。例如，如果从上一节中提取text-target，并使用draw-bitmap将其复制到另一个DC，则会传输黑色矩形和蓝色文本，背景单独保留：

(define new-target (make-bitmap 100 30))
(define dc (new bitmap-dc% [bitmap new-target]))
(send dc set-pen "black" 1 'transparent)
(send dc set-brush "pink" 'solid)
 
(send dc draw-rectangle 0 0 100 30)
(send dc draw-bitmap text-target 0 0)

关于位图的像素绘制的信息（而不是“无”）是位图的alpha通道（alpha channel）。不是所有的DC都保持alpha通道，但默认情况下，使用make-bitmap创建的位图保持alpha通道。使用读取位图加载的位图通过位图的alpha通道保持图像文件的透明度。

alpha通道不是全部或全部。例如，当边文本被draw-text消除混叠时，像素部分透明。当像素转移到另一个DC时，部分透明的像素在一个称为alpha混合（alpha blending）的过程中与目标像素混合。此外，DC具有应用于所有绘图操作的alpha值；alpha值1.0对应于实体绘图，alpha值0.0使绘图无效，并且介于两者之间的值使绘图透明。

例如，在调用draw-bitmap之前，将DC的alpha设置为0.25会导致“你好，世界！“位图与目标图像混合时的四分之一强度：

(send dc erase)
(send dc draw-rectangle 0 0 100 30)
(send dc set-alpha 0.25)
(send dc draw-bitmap text-target 0 0)

1.7 剪辑

除了缓和绘图操作的不透明度之外，DC还有一个剪切区域（clipping region），它将所有绘图限制在该区域内。在最简单的情况下，剪切区域对应于闭合路径，但它也可以是并集、交集、减法或排斥或两条路径中的一条。

例如，可以将剪切区域设置为三个圆以剪切矩形的图形（0.25 alpha仍然有效）：

(define r (new region%))
(let ([p (new dc-path%)])
  (send p ellipse 0 0 35 30)
  (send p ellipse 35 0 30 30)
  (send p ellipse 65 0 35 30)
  (send r set-path p))
(send dc set-clipping-region r)
(send dc set-brush "green" 'solid)
(send dc draw-rectangle 0 0 100 30)

裁剪区域可以看作是一个方便的替代路径填充或绘制点画。相反，点画可以被看作是一种方便的替代方法，可以剪切重复调用的draw-bitmap。

但是，将区域与具有渐变的pen%对象组合不仅仅是一种方便，因为它允许我们以其他方式无法绘制的组合方式绘制形状。为了说明这一点，这里有一些代码用它下面的反射来绘制文本。

; 首先计算要绘制的文本的大小，
; 使用我们从未绘制过的小位图。
(define bdc (new bitmap-dc% [bitmap (make-bitmap 1 1)]))
(define str "Racketeers, ho!")
(define the-font (make-font #:size 24 #:family 'swiss
                            #:weight 'bold))
(define-values (tw th)
  (let-values ([(tw th ta td)
                (send dc get-text-extent str the-font)])
    (values (inexact->exact (ceiling tw))
            (inexact->exact (ceiling th)))))
 
; 现在，我们可以创建一个大小正确的位图
; 来实际绘制并启用平滑。
(send bdc set-bitmap (make-bitmap tw (* th 2)))
(send bdc set-smoothing 'smoothed)
 
; 接下来，构建包含文本大纲的路径
(define upper-path (new dc-path%))
(send upper-path text-outline the-font str 0 0)
 
; 接下来，构建包含文本镜像轮廓的路径
; 轮廓的路径
(define lower-path (new dc-path%))
(send lower-path text-outline the-font str 0 0)
(send lower-path transform (vector 1 0 0 -1 0 0))
(send lower-path translate 0 (* 2 th))
 
 
 
 
 
 
 
; 此助手接受路径，将bdc的剪切区域设置为路径（但以区域形式），
; 然后在整个位图上绘制一个大矩形。在每次调用绘制路径之前，将对
; 画笔进行不同的设置，以便绘制文本，然后绘制阴影。
 
(define (draw-path path)
  (define rgn (new region%))
  (send rgn set-path path)
  (send bdc set-clipping-region rgn)
  (send bdc set-pen "white" 1 'transparent)
  (send bdc draw-rectangle 0 0 tw (* th 2))
  (send bdc set-clipping-region #f))
 
; 现在我们用实心画笔绘制上一条路径
(send bdc set-brush "black" 'solid)
(draw-path upper-path)
 
 
 
 
; 为了绘制阴影，我们在阴影所在的位图部分
; 设置了一个具有线性渐变的画笔。
(define stops
  (list (list 0 (make-color 0 0 0 0.4))

1.8 可移植性和位图变体

绘图效果不能完全移植到跨平台、跨实现dc<%>的不同类或不同类型的位图。字体和文本，尤其是，可以在不同的平台和DC类型之间变化，但也可以通过绘制一条线触摸到精确的像素集。

不同类型的位图可以产生不同的结果：

  • 绘制由make-bitmap（或从bitmap%实例化）生成的位图时，以最一致的方式跨平台绘制。

  • make-platform-bitmap生成的位图的绘制尽可能使用平台特定的绘制操作。然而，在Windows上，由make-platform-bitmap生成的位图没有alpha通道，它使用的资源比make-bitmap生成的资源更多（由于系统范围内的每个进程的GDI限制）。

    作为平台特定差异的一个示例，文本在Mac OS上默认使用亚像素消除混叠进行平滑处理，而make-bitmap结果中的文本平滑处理仅使用灰色。线条或曲线绘图可能与“生成位图”生成的位图中的像素不同，并且位图缩放可能有所不同。

    在Windows下处理GDI限制的一种可能方法是将 make-platform-bitmap的结果绘制出来，然后将绘图的内容复制到make-bitmap的结果中。该方法保留了make-platform-bitmap的绘制结果，但只保留了绘制过程中受限的资源。

  • 从racket/gui/base绘制到make-screen-bitmap生成的位图使用与绘制到canvas%实例相同的平台特定绘图操作。make-screen-bitmap生成的位图使用与Windows或Mac OS上make-platform-bitmap相同的特定于平台的图形，但可能是按比例缩放的，并且可能对Unix上的x11服务器进行缩放或敏感。

    在Mac OS上，当主屏幕处于视网膜模式（创建位图时），位图也会进行内部缩放，以便一个绘图单元使用两个像素。同样，在Windows或Unix上，当主显示器的文本比例在操作系统级别配置时（请参见(part ("(lib scribblings/gui/gui.scrbl)" "显示分辨率（display-resolution）"))），位图将进行内部缩放，其中常用配置将绘图单元映射到1.25、1.5或2像素。

    在将图像传输到屏幕之前，或者由于某些其他原因需要与屏幕绘图保持一致时，使用make-screen-bitmap作为屏幕外缓冲区。

  • 由racket/gui/base的make-bitmap in canvas%在画布中生成的位图与make-screen-bitmap的位图类似，但在Mac OS上，该位图针对绘制到屏幕进行了优化（通过利用系统API，反过来又可以利用图形硬件）。

    将make-bitmap in canvas%用于与make-screen-bitmap类似的用途，特别是在稍后将位图绘制到已知目标画布时。
